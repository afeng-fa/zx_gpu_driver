# 问题分析与踩坑记录

## 一、驱动选择踩坑

### 错误假设
最初假设麒麟系统使用 arise 驱动，直接在 arise.ko 上添加 PCI ID `0x3A04`。

### 实际情况
- arise.ko 的 PCI ID 表只有 `3D00` 系列
- `3A04` 是 CHX002 家族，与 Arise 是**完全不同的 GPU 架构**
- 修改 PCI ID 表后驱动加载成功，但 probe 阶段崩溃（MMIO 偏移 0x8E080 超出 BAR0 512K 范围）

### 正确驱动
`zx.ko`（兆芯官方驱动），modinfo 确认包含 `pci:v00001D17d00003A04` alias。

## 二、GLVND 问题踩坑

### 错误操作
强制设置环境变量：
```
GBM_BACKEND=zx_gbm
__EGL_VENDOR_LIBRARY_FILENAMES=/usr/share/glvnd/egl_vendor.d/10_zx.json
LIBGL_ALWAYS_SOFTWARE=0
```

### 后果
GLVND fallback 失效，部分软件无法创建 OpenGL Context。

### 正确做法
**不强制指定任何 GPU/EGL/GL backend**，由 GLVND 自动选择。zx 驱动 + Mesa fallback 共存。

## 三、GLX 库冲突

### 问题
`libGLX_zx.so.0` 缺少 `glXGetProcAddressARB` 符号，但排在 ldconfig 的 `libGLX.so.0` 前面，导致 GLX 模块加载失败。

### 错误修复
将 `libglx.so` symlink 到 `libglx_zx.so`。

### 正确修复
**不要 symlink**。保持系统标准 Mesa GLX，zx 内核驱动处理 KMS/DRM，GLX 由 GLVND 自动选择。

## 四、模块持久化

### 问题
`insmod` 加载的模块重启后丢失。

### 正确修复
1. 模块安装到 `/lib/modules/<kernel>/updates/`
2. 运行 `depmod -a`
3. 配置 `/etc/modules-load.d/zx.conf`

## 五、完整依赖链

    PCI 1d17:3a04
        ↓
    zx_core.ko (核心模块)
        ↓
    zx.ko (DRM 驱动)
        ↓
    /dev/dri/card0 (DRM 设备)
        ↓
    ├── zx_drv.so (Xorg 驱动)
    ├── libEGL_zx.so (EGL)
    ├── zx_gbm.so (GBM)
    └── zx_drv_video.so (DRI)
